# Divine Dining - Static Website

## Overview

Welcome to **Divine Dining**, a beautifully crafted responsive website designed to showcase an engaging dining experience. Built with pure HTML, CSS, and JavaScript, this static website serves as a digital platform for a restaurant, displaying essential information including menus, events, contact details, and an enquiry form.

## Purpose

The purpose of the Divine Dining website is to provide potential customers with a seamless experience when exploring menu options, upcoming events, and contact information. The clean and elegant design ensures that all critical information is presented in an easy-to-navigate format, enhancing the overall user experience.

## Features

- **Responsive Layout**: The website shifts fluidly between different screen sizes (desktop, tablet, and mobile), ensuring accessibility for all users.
  
- **Navigation Menu**: A clear and intuitive navigation bar links to all primary sections, allowing users to easily access menu items, events, enquiry forms, and contact information.

- **SEO Integration**: Comprehensive SEO optimizations including keywords, title tags, and meta descriptions for each page.

- **Interactive Elements**: 
  - **Accordions** on the About Us page for improved organization of information.
  - Dynamic content that lists upcoming events interactively using JavaScript.

- **Enquiry and Contact Forms**: User-friendly forms with JavaScript validation and feedback to enhance communication efficiency.

- **Lightbox Gallery**: Clicking on a dish in the menu section opens a larger view in a lightbox for an enhanced visual experience.

- **Google Map Integration**: A fully interactive map embedded on the contact page to help users easily locate the restaurant.

## Installation / How to Run

To set up and run this website locally, please follow these instructions:

1. **Download or Clone this Repository**: 
   - You can download the project as a ZIP file or clone the repository using Git.
   - ```bash
     git clone <repository-url>
     ```

2. **Open HTML Files in a Web Browser**:
   - Simply double-click the `index.html` file, or open it directly from your preferred web browser.
   - For local development, using the **Live Server** extension in Visual Studio Code provides a more dynamic development experience.

3. **Static Website**:
   - No server setup is required as this is a static website. All files can be viewed offline once downloaded.

## Usage Instructions

### Editing Content

- To modify the content displayed on each specific page, simply open the respective HTML files (`menu.html`, `events.html`, `enquiry.html`, `contact.html`, and `aboutus.html`) and make the necessary changes.

### Styling Adjustments

- To modify the website’s styles, navigate to the `css/style.css` file. Here you can adjust colors, fonts, spacing, or layout as needed. The CSS file contains all styling for the website, including:
  - **Typography**: Control the styling of all text elements through headings and body text adjustments.
  - **Menu Items**: Customize how dishes are displayed, including hover effects and image sizes.
  - **Buttons**: Update button styles for interactivity, including hover effects that enhance user engagement.

## Search Engine Optimization (SEO)

### On-Page SEO
- **Keyword Research**: Relevant keywords were identified and incorporated across the site.
- **Title Tags and Meta Descriptions**: Compelling title tags and meta descriptions were created for each page.
- **Header Tags**: Utilized appropriate heading tags (H1, H2, H3) to structure content effectively.
- **Image Optimization**: All images have descriptive filenames and alt text.
- **URL Structure**: Clean and descriptive URLs added across the site.
- **Internal Linking**: Relevant internal links created to support navigation.
- **Mobile-Friendliness**: Ensured the website is fully responsive, improving the mobile experience.

### Off-Page SEO
- Encouraged backlinking strategies to improve site authority.
- Promoted the website on various social media platforms to increase visibility.

### Additional Requests
- **robots.txt**: Created to instruct search engines on which pages to index.
- **sitemap.xml**: Generated to help search engines understand the site structure for better indexing.
- **Speed Optimization**: Optimized images and code minification techniques for faster loading times.
- **Security**: Implemented basic security measures to mitigate potential threats.
